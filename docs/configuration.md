# Configuration

`swig2pyi` is rule-driven. It relies on a JSON configuration file to map C++ features to Python type hints, handle pointers, strip namespaces, and customize output signatures.

## Example Configuration

Below is an example JSON configuration illustrating the available options:

```json
{
    "module_name": "QuantLib",
    "includes": ["/usr/local/include"],
    "type_map": {
        "Integer": "int",
        "Real": "float",
        "Size": "int",
        "std::string": "str"
    },
    "smart_pointers": [
        "std::shared_ptr",
        "boost::shared_ptr"
    ],
    "containers": {
        "std::vector": "list",
        "std::map": "dict"
    },
    "rename_operators": true,
    "namespaces_to_remove": [
        "QuantLib::"
    ],
    "delegate_templates": [
        "Handle",
        "RelinkableHandle"
    ],
    "generic_templates": [
        "std::vector",
        "std::map"
    ],
    "template_arg_counts": {
        "std::vector": 1,
        "std::map": 2
    },
    "parameter_relaxation": {
        "Matrix": "Sequence[Sequence[float]]",
        "Array": "Sequence[float]"
    },
    "pythoncode_signatures": {
        "my_custom_python_helper": "(arg1: int, arg2: float) -> str"
    },
    "skip_functions": [
        "legacy_c_function"
    ],
    "extra_code": [
        "import typing",
        "from typing import Sequence"
    ]
}
```

## Schema Fields

### `module_name`
*   **Type:** `str` (Required)
*   **Description:** The name of the Python module generated by SWIG (e.g., `QuantLib`). Used to resolve relative scopes and strip self-module prefixes.

### `includes`
*   **Type:** `list[str]` (Required)
*   **Description:** A list of inclusion directories passed to the SWIG compiler (`-I` flag) when running in Interface Mode.

### `type_map`
*   **Type:** `dict[str, str]` (Required)
*   **Description:** Direct C++ to Python type mappings (e.g. mapping custom C++ typedef types like `Real` to `float`). Mappings in this dictionary take precedence over automatically discovered wrappers.

### `smart_pointers`
*   **Type:** `list[str]` (Required)
*   **Description:** C++ smart pointer classes to automatically strip (e.g., `std::shared_ptr<T>` resolves to type `T` or `_T` in stubs, delegating access to the underlying class).

### `containers`
*   **Type:** `dict[str, str]` (Required)
*   **Description:** Maps C++ template containers to PEP 484 Python equivalents (e.g. `std::vector` to `list`, or `std::map` to `dict`).

### `rename_operators`
*   **Type:** `bool` (Default: `false`)
*   **Description:** If `true`, renames C++ operator overloads to Python dunder equivalents (e.g., `operator==` to `__eq__`, `operator[]` to `__getitem__`). If `false`, operators are excluded from stubs.

### `namespaces_to_remove`
*   **Type:** `list[str]` (Default: `[]`)
*   **Description:** C++ namespace prefixes to strip from classes and functions (e.g., `QuantLib::` or `std::`).

### `delegate_templates`
*   **Type:** `list[str]` (Default: `[]`)
*   **Description:** Template proxy wrapper classes (such as `Handle<T>`) that should automatically inherit and delegate all public methods of the template argument type `T`.

### `generic_templates`
*   **Type:** `list[str]` (Default: `[]`)
*   **Description:** Specifies C++ template classes that should be translated into Python generic types using `typing.Generic`.

### `template_arg_counts`
*   **Type:** `dict[str, int]` (Default: `{}`)
*   **Description:** Limits the number of template arguments written in stub files. Useful for stripping default allocators or comparators (e.g., keeping only 1 parameter for `std::vector` instead of default C++ allocators).

### `parameter_relaxation`
*   **Type:** `dict[str, str]` (Default: `{}`)
*   **Description:** Relaxes specific parameters to accept wider sequences (e.g., translating `Array` inputs to `Sequence[float]` or `Matrix` to `Sequence[Sequence[float]]` on function definitions).

### `pythoncode_signatures`
*   **Type:** `dict[str, str]` (Default: `{}`)
*   **Description:** Overrides type signatures for functions injected via `%pythoncode` blocks which do not have C++ parser counterparts.

### `skip_functions`
*   **Type:** `list[str]` (Default: `[]`)
*   **Description:** Lists names of C++ functions to completely skip and exclude from the generated stubs.

### `extra_code`
*   **Type:** `list[str]` (Default: `[]`)
*   **Description:** Raw Python lines of code to write directly into the header of the generated stub file (e.g. custom typing imports or type definitions).
